### 프로젝트 개요

YouTube Music의 UI를 참고하여 메인 화면을 클론 코딩한 프로젝트입니다.<br>
HTML, CSS를 활용하여 실제 서비스와 유사한 레이아웃을 구현하는 것을 목표로 하였습니다.<br>
이번 실습에서는 flex를 이용한 기본 레이아웃 구성, 카드 UI, 스크롤 처리 방식을 중점적으로 연습했습니다.

### 화면 구성

Sidebar, Header, Main 영역으로 나누어 구성했습니다.<br>
Sidebar에는 메뉴와 플레이리스트를 배치하고, Header에는 검색창과 사용자 정보를 배치했습니다.<br>
Main 영역에는 텍스트 아이콘과 카드 형태의 콘텐츠 섹션을 배치하여 실제 서비스와 유사한 구조를 만들었습니다.

### HTML 작성 시 신경 쓴 점

시맨틱 태그를 활용하여 각 영역의 역할이 명확하게 드러나도록 구조를 설계했습니다.<br>
Header, Nav, Main, Section 등의 태그를 사용하여 문서 구조를 구분했습니다.<br>
콘텐츠의 의미에 맞게 div를 묶어 가독성과 유지보수를 고려했습니다.

### CSS 작성 시 신경 쓴 점

flex를 활용하여 전체 레이아웃을 구성했습니다.<br>
요소 간 정렬과 간격이 일정하게 유지되도록 margin과 padding을 조절했습니다.<br>
반응형을 고려하여 미디어쿼리를 사용해 화면 크기에 따라 레이아웃이 자연스럽게 변하도록 설계했습니다.

### 구현 중 문제와 해결 

카드 영역이 화면을 넘어갈 때 레이아웃이 깨지는 문제가 발생했고.<br>
flex의 줄바꿈 처리와 overflow-x 설정을 통해 해결했습니다.<br>
Desktop에서는 가로 스크롤 방식으로, Tablet과 Mobile에서는 wrap 방식을 설정하였습니다.

스크롤바가 나타날 때 레이아웃이 흔들리는 문제를 겪었는데,<br>
스크롤바 높이 변화로 인해 발생하는 문제임을 파악하고 공간을 미리 확보하는 방식으로 해결했습니다.

### 실습을 통해 확인한 점

레이아웃을 구성하며 flex의 정렬 방식을 더 명확하게 이해할 수 있었습니다.<br>
카드 섹션을 만들며 지금까지 배웠던 기초 개념들을 더 명확하게 이해할 수 있었습니다.<br>
UI를 구현할 때 작은 디테일이 전체 완성도에 큰 영향을 준다는 것을 느꼈습니다.

### 개선할 점

미디어쿼리 작성에 시간을 많이 할애하지 못해 반응형 디자인이 빈약합니다.<br>
현재는 UI 중심의 구현이기 때문에 실제 음악 재생 기능이나 데이터 연동이 없습니다.<br>
JavaScript를 활용한 상호작용을 더 추가하면 더 완성도 높은 프로젝트가 될 수 있을 것 같습니다.
